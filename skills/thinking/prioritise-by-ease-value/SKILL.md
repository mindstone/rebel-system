# Prioritise by Ease/Value

Rank ideas by multiplying ease and value scores (1-3 each) to identify high-priority items.

Works well informally—you don't need spreadsheets, just use the ease/value lens conversationally.


## See also

- [interview-me-to-look-for-AI-automations.md](../companyos/interview-me-to-look-for-AI-automations/SKILL.md) - uses this framework to rank automation opportunities
- [Greg Detre's ease/value article](https://www.makingdatamistakes.com/ease-value/) - detailed background including multi-rater scoring


## Process

1. **Score each idea on Value (1-3, higher = better)**
   - High (3): Significant impact, CEO would care, transformative
   - Medium (2): Noticeable benefit, helps achieve goals
   - Low (1): Minor benefit, small impact

2. **Score each idea on Ease (1-3, higher = better)**
   - High (3): Quick, straightforward, low risk, done before
   - Medium (2): Some complexity, moderate time/unknowns
   - Low (1): Slow, laborious, novel, high uncertainty

3. **Calculate Priority = Value × Ease** (scale 1-9)

4. **Sort by priority, focus discussion on top 3-5 items**

**Key principle:** High value + Easy (3×3 = 9) always wins. High value + Hard (3×1 = 3) ties with Low value + Easy (1×3 = 3), so use judgment.


## Example

| Idea                         | Value | Ease | Priority | Notes                             |
| ---------------------------- | ----- | ---- | -------- | --------------------------------- |
| Calendar→Slack weekly update | 3     | 3    | 9        | 15 min/week saved, MCPs available |
| Notion database checks       | 2     | 3    | 6        | Easy query, moderate value        |
| Automated board reports      | 3     | 1    | 3        | High value but complex logic      |
